- IoC 컨테이너 개념
```java
+-------------------------------------------------------+
|                        애플리케이션 코드                   |
|                                                       |
|   개발자가 작성한 코드로, 빈을 사용하는 곳               |
+-------------------------------------------------------+
                               ↓
+-------------------------------------------------------+
|                    Spring IoC 컨테이너                  |
|                                                       |
|   빈의 생성, 관리, 의존성 주입, 빈의 생명주기 관리 등을 담당 |
|   (Bean Factory 또는 Application Context)              |
+-------------------------------------------------------+
   /              |                     \
  /               |                      \
 /                |                       \
↓                 ↓                        ↓
+-----------+     +-----------+        +-----------+
|   빈     |     |   빈     |        |   빈     |
|  (객체)  |     |  (객체)  |        |  (객체)  |
+-----------+     +-----------+        +-----------+
  (생성 및          (생성 및             (생성 및
   관리됨)            관리됨)               관리됨)

```
- IoC 컨테이너는 제어권을 가지고 있기 때문에, 개발자는 빈을 직접 생성하고 관리할 필요 없이 빈의 설정과 관계를 정의하면 컨테이너가 이 설정을 기반으로 빈을 생성하고 의존성을 주입
-  개발자는 더 추상적인 레벨에서 애플리케이션을 설계하고 개발 가능
